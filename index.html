IP-GUARD在计算机的安全模式下，加密客户端能否正常运行？

发表于 2020-06-17 分类于 IP-GUARD Valine：

在安全模式下，加密客户端也能正常运行、正常加解密文件。

IP-GUARD安全密码有什么作用呢？

发表于 2020-06-17 分类于 IP-GUARD Valine：

1、为防止他人使用此客户端非法解密文件，客户端应该要设置一个密码对文件解密或者外发进行控制，在每次解密或者外发时需要输入此密码；

2、客户端离线后，使用安全密码登陆离线授权模式后才能正常使用离线加解密权限，增强客户端离线状态下加密文档的安全保护。

IP-GUARD用授权软件直接打开文档不做修改，此时能否自动加密？

发表于 2020-06-17 分类于 IP-GUARD Valine：

​ 如果打开文档就直接加密的话，会导致软件打开速度慢，且软件的配置文件、非授权软件的文件、系统文件都会被加密，这会影响到非授权软件的运行，所以不会去实现这个功能。

ip-guard服务器不能上外网，补丁管理该怎样使用？

发表于 2020-06-17 分类于 IP-GUARD Valine：

服务器不能上外网，补丁管理该怎样使用？

如果服务器所在的计算机不能访问公网，此时需要将补丁管理模块迁移到能访问公网的计算机上执行，才能从微软的网站上下载补丁检测文件和补丁文件。 具体的步骤如下：

1.停止服务器上OUpdate的运行：在系统服务中(在运行中输入“services.msc”打开服务管理器)停止OCULAR V3 Update,并将其启动类型改为”手动”。

2.转移补丁管理，在能连上互联网的计算机上（该计算机也能和服务器机器正常通讯）：首先建立一个目录，将服务器上的OUpdate.exe和Update.ini文件复制到这个目录下。修改

Update.ini文件并保存：

[Config] IP=服务器的IP地址

3.安装补丁管理服务，在这台计算机上运行命令提示符，将路径切换到该目录下（例如

OUpdate.exe和Update.ini所在目录为d:\ipguard3\oupdate，则在命令提示符下首先输入

“d:”回车，然后键入“cd ipguard3\oupdate”），输入“OUpdate.exe –i”命令安装补

丁管理服务，然后进入系统的服务窗口就能见到”OCULAR V3 Update”，启动该服务即可。

ip-guard如果服务器 IP 地址或机器名变更之后对客户端或控制台会有影响吗？

发表于 2020-06-17 分类于 IP-GUARD Valine：

1、 服务器的 IP 地址或机器名变更包括两种情况：一种是服务器所在的计算机 IP 地址或机器名发生了变化，另一种情况是服务器迁移到别的计算机上改变了 IP 地址或机器名；第一种情况如果服务器开启了主动轮询功能（在控制台“工具”->“选项”->“连接设置”里的主动轮循项，默认是启动的）客户端与服务器的通讯是没有影响的，第二种情况下，为了确保服务器与客户端能正常连接，最好设置服务器的检验码为原来的检验码，并启动主动轮询功能。

2、 当服务器 IP 地址或机器名变更后，只需在登陆控制台时输入新的服务器 IP 或机器名就可以了。针对整个网络，在控制台设置策略客户端配置-自定义 serverspare=新IP，

ip-guard文档加密后无小锁标志

发表于 2020-06-17 分类于 IP-GUARD Valine：

加密锁图标是用来标记显示加密文档，而非加密文件和外发文件都不会显示。

  1. 确定文件是加密文件，可以通过打开本地扫描来扫描指定的文件，查看是否有小锁标志。如果有但是在桌面和资源管理器中查看文件无小锁标志，可能被保护软件处理了或被用户恶意修改过。

  2. 确认是否设置了隐藏加密锁图标策略

  3. 查看C:\Windows\System32\SDFileIcon2.dll（备份文件baksdfi2.sys）是否存在，如果不存在，则查看杀毒软件日志，是否有查杀记录，若无记录，需要重装客户端。

  4. 若有记录，使用Procexp工具查看Explorer.exe进程中是否有SDFileIcon2.dll或SDFileIcon64.dll模块。如果没有，注册COM组件

开始-运行：输入cmd打开dos命令行，输入regsvr32.exe sdfileicon2.dll（或者 “regsvr32 sdfileicon64.dll” (x64)），注册模块成功后，重启explorer.exe
